1.1 Reading a Delimited File with read_delim()

We’ll start by reading in the household_power_consumption.txt dataset from the UC Irvine Machine Learning Repository https://archive.ics.uci.edu/dataset/235/individual+household+electric+power+consumption. In this case we can use the menus to read in the data we saved locally. If you open the raw file, you’ll notice the values are separated by semicolons (;) rather than commas — this is a semicolon-delimited file.

If we go to File -> Import Dataset we can choose to import a “From Text” data using base R or the readr method. We chose the readr way and checked out the default method. We had to change our delimiter to a “;” and the data should read in.

The code from the import menu is produced here.

library(readr)
household_power_consumption <- read_delim("~/Downloads/household_power_consumption.txt",

The ? character is being used to represent missing values in this dataset, which confuses readr. We can fix this using the na = argument:

library(readr)
household_power_consumption <- read_delim("~/Downloads/household_power_consumption.txt", 
    delim = ";", escape_double = FALSE, trim_ws = TRUE, na = c("", "NA", "?"))

1.2 Tibble vs. Data Frame

Before we dive into manipulation, let’s quickly note that read_delim()/any reading functions from readr package return tibbles, not plain data frames. Tibbles are a tidyverse-friendly version of a data frame with some helpful differences:

  • They print only the first 10 rows and as many columns as fit on screen.
  • They never changes the type of the inputs (e.g. never converts strings to factors!)
  • They never changes the names of variables
  • They never create row names.

Let’s look at the differences in the output when we type flights (tibble) in the console versus typing mtcars (data frame) in the console.

You can coerce a regular data frame to a tibble with as_tibble(). Let’s use as_tibble() on the mtcars data set. Save this new data set as mtcars_tibble. Check your environment to see if it worked!
