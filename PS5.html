tapply(): Grouped Summaries

tapply() is a member of the apply family that often flies under the radar. It lets you apply a function to subsets of a vector, where the subsets are defined by a grouping factor — essentially a base R version of group_by() |> summarize().

The syntax is:

tapply(X, INDEX, FUN, ...)

where X is the vector of values, INDEX is the grouping factor (or list of factors), and FUN is the function to apply.

Your turn: Using tapply() on the penguins data, compute the mean body mass (in grams) for each combination of species and island.

tapply(X = penguins$body_mass_g,
       INDEX = list(penguins$species, penguins$island),
       FUN = mean,
       na.rm = T)

map_dfr() and Combining Results

Now let’s introduce a particularly useful function from purrr: map_dfr(). One nice feature of purrr over base R’s apply family is that we have direct control over the output format. The dfr in map_dfr() stands for “data frame, row-bound” — it applies a function to each element of a list and automatically stacks the results into a single tibble by binding rows.

Because we’re working with data frames in our list, it may (or may not) be logical that we want to combine everything into one working data frame. We’ll use the flights_list from above.

For this exercise, work through the following three steps:

– Use map_dfr() to calculate the column means for all numeric variables in each data frame in flights_list.

– Create a new variable called group that identifies which list element each row came from — values should be 1 for UA and 2 for DL.

– Pivot the resulting data set into a longer form that would be better suited for plotting or modeling (one row per carrier–variable combination).

# Helper Function

get_numeric_means <- function(df) {
  df |> 
    select(where(is.numeric)) |>
    summarise(across(everything(), \(x) mean(x, na.rm = T)))
}

map_dfr(flights_list, get_numeric_means) |>
  mutate(group = 1:2) |>
  pivot_longer(cols = 1:4,
               names_to = "variable",
               values_to = "mean_value")

Requirements:

  • The function should work in a pipeline.
  • Use stop() to throw an informative error if the input is not a data frame.
inherits(penguins, "vector")

scale_numeric_cols <- function(.df){
  if(!inherits(.df, "data.frame")) {
    stop("Input must be a data frame or a tibble")
  }
  
  .df |>
    mutate(across(where(is.numeric), \(x) (x - mean(x, na.rm = T))/ sd(x, na.rm = T)))
}
